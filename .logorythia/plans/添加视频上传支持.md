# 摘要
在现有图像支持架构上做镜像扩展，为 SwiftOpenAI 库添加 `video_url` 视频上传支持。Kimi/豆包/通义千问/智谱GLM 均使用统一结构 `{ "type": "video_url", "video_url": { "url": "..." } }`，可选 `fps` 字段。

# 修改一：MessageConvenience.swift — 模型层扩展

在 `UserMessageParam.Content.ContentPart` 和 `ToolMessageParam.Content.ContentPart` 两个枚举中同步新增 `video` case，并添加对应的 `VideoContent` / `VideoURL` 类型。

具体位置与操作：
- `file:///Users/zhiyang/开发/Packges/SwiftOpenAI/Sources/SwiftOpenAI/Models/MessageConvenience.swift:<57>-<159>`：User 的 `ContentPart` 枚举新增 `.video(VideoContent)`；`init(from decoder:)` 增加 `"video_url"` 分支；`encode(to encoder:)` 增加 video 分支。在其 `ImageContent` 同级位置新增 `VideoContent` struct，含 `type: String = "video_url"` 与 `videoUrl: VideoURL`。新增 `VideoURL` struct，含 `url: String` 与 `fps: Double?`；提供 `init(url: String, fps: Double? = nil)` 与 `init(videoData: Data, fps: Double? = nil)`，后者做 Base64 编码并默认 MIME 为 `video/mp4`。
- `file:///Users/zhiyang/开发/Packges/SwiftOpenAI/Sources/SwiftOpenAI/Models/MessageConvenience.swift:<323>-<421>`：Tool 的 `ContentPart` 做完全镜像修改（Tool 的 `VideoContent`/`VideoURL` 可独立定义，类型名与 User 相同但位于 ToolMessageParam 命名空间下，保持与现有 Image 模式一致）。
- `file:///Users/zhiyang/开发/Packges/SwiftOpenAI/Sources/SwiftOpenAI/Models/MessageConvenience.swift:<425>-<479>` 之后：新增 `ContentPartVideoParam` struct（镜像 `ContentPartImageParam`），含 `videoUrl: VideoURL`，`VideoURL` 结构与 User 的一致。
